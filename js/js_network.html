📚 네트워크 데이터의 본질
🎯 핵심 · 모든 네트워크 데이터는 두 개의 표로 환원됩니다 — 노드 표(Nodes)와 엣지 표(Edges). 이 두 가지만 만들어지면 어떤 라이브러리에서든 시각화 가능.
✨ 비유 · "사람 명단(노드)" + "누가 누구와 통화했는지 통화 기록(엣지)" — 둘만 있으면 통화 관계망이 그려집니다.

1️⃣ 노드(Node) 표 — "점"의 명단

각 행 = 하나의 노드(사람·시도·키워드 등). id는 고유해야 합니다.

id (필수·고유) label (화면 표시) group (색·범주) value (크기, 선택)
서울서울특별시시도34215
경기경기도시도58432
차대차차대차 사고유형142350

2️⃣ 엣지(Edge) 표 — "연결"의 명단

각 행 = 한 쌍의 연결. from/to는 위 노드 표의 id와 정확히 일치해야 합니다.

from (시작 id) to (끝 id) value (선 굵기·가중치) label (선 라벨, 선택)
서울차대차2580025,800건
서울차대사람68006,800건
경기차대차4410044,100건
💡 핵심 규칙: 엣지 표의 from·to 값은 노드 표의 id와 정확히 같아야 합니다. 한 글자라도 다르면(예: "서울"과 "서울시") 그 노드가 자동 생성되거나 연결이 끊깁니다.

3️⃣ Excel에서 작성하는 가장 쉬운 방법

📂 my_network.xlsx
├── Sheet1: nodes        ← 노드 표
│     id    | label       | group  | value
│     서울  | 서울특별시  | 시도   | 34215
│     ...
└── Sheet2: edges        ← 엣지 표
      from  | to          | value
      서울  | 차대차      | 25800
      ...

① Excel에서 두 시트를 만들고 위처럼 입력 → ② 각각을 CSV로 저장(다른 이름으로 저장 → CSV UTF-8) → ③ JS에서 읽기.

4️⃣ CSV → JS 로딩 코드 (PapaParse)

// CDN 추가:
// <script src="https://cdn.jsdelivr.net/npm/papaparse@5"></script>

Promise.all([
    fetch('nodes.csv').then(r => r.text()),
    fetch('edges.csv').then(r => r.text())
]).then(([nodeCsv, edgeCsv]) => {
    const nodes = Papa.parse(nodeCsv, { header:true, dynamicTyping:true }).data;
    const edges = Papa.parse(edgeCsv, { header:true, dynamicTyping:true }).data;
    new vis.Network(div,
        { nodes: new vis.DataSet(nodes), edges: new vis.DataSet(edges) },
        options);
});

6️⃣ 원본 데이터 → 노드/엣지 표 만들기 (집계 패턴)

실제 사고 원본은 보통 "한 행 = 한 사고" 형태입니다. 이걸 어떻게 노드/엣지로 바꿀까요? 

📂 원본 (accidents.xlsx)        ← 한 행 = 한 사고
   사고ID | 시도   | 유형     | 사망자 | ...
   001    | 서울   | 차대차   | 0      |
   002    | 서울   | 차대사람 | 1      |
   003    | 경기   | 차대차   | 0      |
   ...

🔧 Excel 피벗테이블 또는 SUMIFS / COUNTIFS / 파이썬 groupby 로 집계:

📂 edges.csv     ← 시도와 유형의 동시 발생 빈도
   from  | to       | value
   서울  | 차대차   | 25800
   서울  | 차대사람 | 6800
   경기  | 차대차   | 44100
   ...

📂 nodes.csv     ← 각 노드의 합계
   id        | group | value
   서울      | 시도  | =SUM(시도='서울' 행의 모든 사고)
   차대차    | 유형  | =SUM(유형='차대차' 행의 모든 사고)
   ...
💡 Excel의 피벗테이블로 시도 × 유형 교차표를 만든 뒤, "수식 → 셀 → from/to/value 형태로 복사"하면 엣지 표가 됩니다. Python이면 df.groupby(['시도','유형']).size().reset_index(name='value') 한 줄.

7️⃣ 추가 옵션 컬럼 — 풍부하게 만들기

컬럼명 의미 예시 효과
color노드/엣지 색상 직접 지정"#3b82f6" → 파란색
shape노드 모양dot · box · diamond · star
title마우스 hover 툴팁"서울 사고 34,215건"
arrows방향 표시 (엣지)"to" → 화살표
dashes점선 (엣지)true → 점선
⚠️ 흔한 실수 4가지
  • id 중복 — 노드 표에 같은 id가 두 번 들어가면 에러
  • 이름 불일치 — 엣지의 "서울" vs 노드의 "서울특별시" → 연결 안 됨
  • 빈 값 — value 컬럼에 공백/빈 셀이 있으면 NaN 오류
  • 인코딩 — CSV는 반드시 UTF-8로 저장(엑셀에서 "CSV UTF-8" 선택)
vis-network 기본 — 노드 + 엣지 + 물리엔진
📚 네트워크 시각화란?
🎯 목적 · "무엇이 무엇과 연결돼 있는가?" — 관계 구조를 점(노드)과 선(엣지)으로 표현. 표·차트로 안 보이는 패턴이 드러남.
✨ 특징 · vis-network는 물리엔진을 내장 — 노드끼리 밀어내고 엣지가 잡아당겨 자동으로 균형 잡힘. 드래그로 이동도 가능.
🚦 교통사고 사례
  • "5건의 사망사고가 같은 운전자/차량/교차로와 연결?" 같은 공통 인자 추적
  • 차량 부품 결함 → 사고 연쇄 (제조사-부품-차종-사고)
  • 음주운전 가해자-피해자 관계망
⚠️ 주의 · 노드 50개 넘으면 "털뭉치(hairball)" 현상 → 필터링·계층 분리 필수.
// CDN: vis-network/standalone/umd/vis-network.min.js
const nodes = new vis.DataSet([
    { id:1, label:'A' }, { id:2, label:'B' }, { id:3, label:'C' }
]);
const edges = new vis.DataSet([
    { from:1, to:2 }, { from:2, to:3 }, { from:3, to:1 }
]);
const network = new vis.Network(
    document.getElementById('net1'),
    { nodes, edges },
    { physics: { stabilization: true } }
);
💡 physics가 기본 활성화 → 노드들이 자동으로 자리잡습니다. 드래그로 직접 옮길 수도 있어요.
⭐ 양분 네트워크 — 시도 ↔ 사고유형
📚 양분(Bipartite) 네트워크란?
🎯 목적 · 두 종류의 그룹(예: 시도 vs 사고유형) 사이에 "어디가 어떤 유형이 많은지"를 한눈에 보여줌. 표·히트맵보다 직관적.
✨ 특징 · 노드 크기 = 합계, 엣지 굵기 = 두 그룹 사이 빈도. 색을 그룹별로 다르게.
🚦 교통사고 사례
  • "경기-차대차" 엣지가 가장 굵음 → 경기도가 차대차 사고가 가장 많은 곳
  • 차종(승용·승합·이륜) ↔ 사고유형 매칭 → 이륜차는 차대차 비중이 적고 단독 비중이 큼 등 패턴 발견
  • 가해 연령대 ↔ 위반항목 → "20대-과속" "70대-신호위반" 같은 강한 연결 강조
⚠️ 주의 · 양쪽 그룹 라벨이 헷갈리지 않도록 색 구분 필수. 엣지가 너무 많으면 곡선 옵션으로 겹침 완화.

계층(Tree) 레이아웃 — 위계가 분명한 구조
📚 계층 레이아웃이란?
🎯 목적 · 부모-자식, 상위-하위가 분명한 데이터(조직도·분류체계)를 한 방향으로 정렬해 위계가 한눈에 보이도록.
✨ 특징 · 물리엔진 끄고 알고리즘으로 깔끔하게 배치. 방향(상하·좌우)을 자유롭게 선택.
🚦 교통사고 사례
  • "전국 → 4개 권역(수도권·영남·호남·중부) → 17개 시도" 행정 위계
  • "사고 분류 체계: 인적요인 → 운전자/보행자 → 세부 위반항목"
  • "보험 청구 흐름: 신고 → 평가 → 합의/소송 → 종결"
⚠️ 주의 · 사이클(순환)이 있는 그래프는 계층 표현 불가. 그땐 force-directed가 적합.
options = {
    layout: {
        hierarchical: {
            direction: 'UD',           // UD/DU/LR/RL
            sortMethod: 'directed',
            levelSeparation: 110,
            nodeSpacing: 150
        }
    },
    physics: false   // 계층에서는 끄는 것이 깔끔
};
D3 Force-Directed — 직접 시뮬레이션 구성
📚 D3 Force-Directed란?
🎯 목적 · 노드끼리 밀어내고(charge), 엣지로 잡아당기는(link) 물리 시뮬레이션을 SVG로 직접 구성. 커스터마이즈 자유도 최고.
✨ 특징 · vis-network보다 코드가 길지만 디자인 100% 제어. 학술 논문급 시각화도 가능.
🚦 교통사고 사례
  • 3그룹(시도·유형·시간대) 다중 카테고리 관계망 — 비슷한 항목끼리 자연스럽게 군집
  • 차량부품-제조사-결함 코드 네트워크
  • 저자-인용 네트워크 같은 학술적 활용
⚠️ 주의 · D3는 학습 곡선이 가파름. 빠르게 결과만 필요하면 vis-network이 낫고, 디자인이 핵심이면 D3.
const sim = d3.forceSimulation(nodes)
    .force('link',   d3.forceLink(links).id(d => d.id).distance(80))
    .force('charge', d3.forceManyBody().strength(-300))
    .force('center', d3.forceCenter(W/2, H/2));

sim.on('tick', () => {
    link.attr('x1', d => d.source.x).attr('y1', d => d.source.y)
        .attr('x2', d => d.target.x).attr('y2', d => d.target.y);
    node.attr('transform', d => `translate(${d.x},${d.y})`);
});
Cytoscape.js — 그래프 분석 라이브러리 (Dagre 레이아웃)
📚 Cytoscape란?
🎯 목적 · 시각화 + 그래프 알고리즘(최단경로·중심성·커뮤니티 검출)을 한 라이브러리에서. 분석이 핵심일 때.
✨ 특징 · 다양한 레이아웃(dagre·cose·circle·grid·concentric·breadthfirst) 즉시 전환. 셀렉터 문법으로 노드/엣지 조회·필터.
🚦 교통사고 사례
  • 도로 네트워크에서 사고지점 → 가장 가까운 병원까지 최단경로 계산
  • 교차로 노드 중심성 분석 → "어느 교차로가 가장 중요한 길목인가"
  • 사고 데이터로 동시발생 패턴 군집 분석
⚠️ 주의 · 시각화만 필요하면 vis-network가 더 가벼움. Cytoscape는 분석 기능을 위해 선택.
cytoscape({
    container: document.getElementById('net5'),
    elements: [
        { data:{ id:'a' } }, { data:{ id:'b' } },
        { data:{ source:'a', target:'b' } },
    ],
    style: [
        { selector:'node', style:{ 'background-color':'#4338ca', 'label':'data(id)' } },
        { selector:'edge', style:{ 'curve-style':'bezier', 'target-arrow-shape':'triangle' } }
    ],
    layout: { name:'dagre' }
});
⭐ 인터랙티브 조작 — 노드/엣지 추가·삭제·이웃 강조
📚 인터랙티브 그래프 편집이란?
🎯 목적 · 정적 그림이 아니라 독자가 그래프를 직접 수정·탐색할 수 있게 만들기. 데이터 입력 도구로도 활용 가능.
✨ 특징 · vis-network DataSet은 add()/remove()/update()만 호출하면 화면이 자동 갱신. 이웃 강조는 getConnectedNodes()로.
🚦 교통사고 사례
  • 수사관이 "이 사람도 같은 사고에 연관됐다" 새 노드 추가하며 관계망 구성
  • 특정 운전자 노드 클릭 → 그가 관여한 사고/차량/피해자만 강조
  • 의심 노드를 삭제(필터)하며 패턴이 어떻게 변하는지 시뮬레이션
⚠️ 주의 · 변경된 데이터는 화면에만 남음. 영구 저장하려면 서버 API 또는 localStorage를 함께 써야 함.
// ── 핵심 API ──
// 노드 추가
nodes.add({ id, label, color });

// 엣지 추가
edges.add({ id:'e1', from:1, to:2 });

// 선택된 항목 가져오기
const sel = network.getSelection();   // { nodes:[...], edges:[...] }

// 이웃 노드 가져오기 + 강조
const neighbors = network.getConnectedNodes(nodeId);
nodes.update(allNodes.map(n => ({

⭐ 의미연결망(Semantic Network) — 텍스트 속 단어 관계 그리기
📚 의미연결망이란?
🎯 목적 · 워드클라우드가 "어떤 단어가 자주 나오는가"라면, 의미연결망은 한 걸음 더 나아가 "어떤 단어들이 함께 나오는가"를 보여줍니다. 키워드 사이의 관계 구조가 드러남.
✨ 핵심 원리 — 동시출현(co-occurrence)
  • 같은 문장(또는 단락·기사) 안에 두 단어가 함께 등장하면 → 두 단어 사이에 엣지 +1
  • 여러 문장에서 자주 같이 나올수록 엣지가 두꺼워짐
  • 중심성이 높은 단어(연결이 많은 단어) = 담론의 핵심 키워드
🚦 교통사고 사례
  • 경찰 조서 1,000건 → "음주" "야간" "신호위반"이 강하게 연결되면 야간 음주의 신호위반 패턴이 담론에 형성
  • 국민 민원·SNS 분석 → "어린이" "보호구역" "과속" 군집 → 교육·정책 우선순위
  • 뉴스 기사 키워드 → 사건 보도 프레임 분석
  • 자유서술형 설문(사고 원인) → 응답자가 인식하는 원인 구조
⚠️ 주의 · 동시출현 ≠ 인과. 또 불용어(은·는·이·가)와 너무 흔한 단어(예: "사고")는 모든 단어와 연결되어 의미가 흐려지므로 빈도 상위·하위 컷오프가 필요.

🛠️ 의미연결망 만드는 4단계

  1. 전처리: 텍스트를 문장 단위로 쪼개고, 불용어(은·는·이·가) 제거, 명사·형용사만 추출
  2. 빈도 계산: 단어별 등장 횟수 → 노드의 크기(=value)
  3. 동시출현 카운트: 같은 문장에 함께 나온 단어 쌍의 빈도 → 엣지의 굵기(=value)
  4. 시각화: 노드/엣지 표를 vis-network 같은 force-directed에 입력

💻 코드 — 텍스트 → 의미연결망 자동 생성

// 1) 문장 단위로 쪼갠 뒤 단어 추출
const sentences = text.split(/[.!?\n]+/);
const stopWords = new Set(['은','는','이','가','을','를','의','에','와','과','도']);
function tokens(s) {
    return s.split(/[\s,·\-—()【】\[\]"'?!]+/)
            .filter(w => w.length >= 2 && !stopWords.has(w));
}

// 2) 단어 빈도 + 동시출현 카운트
const freq = {}, pair = {};
sentences.forEach(s => {
    const ws = [...new Set(tokens(s))];        // 같은 문장 내 중복 제거
    ws.forEach(w => { freq[w] = (freq[w]||0) + 1; });
    for (let i=0; i<ws.length; i++)
        for (let j=i+1; j<ws.length; j++) {
            const key = [ws[i], ws[j]].sort().join('|');
            pair[key] = (pair[key]||0) + 1;
        }
});

// 3) 상위 N개 단어만 노드로 (희귀어 제거)
const topWords = Object.entries(freq).sort((a,b)=>b[1]-a[1]).slice(0, 30);
const wordSet  = new Set(topWords.map(w => w[0]));
const nodes = topWords.map(([w,c]) => ({ id:w, label:w, value:c }));

// 4) 두 단어 모두 상위에 포함되고, 동시출현 ≥ 2 인 엣지만
const edges = Object.entries(pair)
    .filter(([k,v]) => v >= 2 && k.split('|').every(w => wordSet.has(w)))
    .map(([k,v]) => { const [a,b] = k.split('|'); return { from:a, to:b, value:v }; });

new vis.Network(div, { nodes:new vis.DataSet(nodes), edges:new vis.DataSet(edges) },
    { nodes:{ shape:'dot', scaling:{min:10,max:50,label:{min:11,max:20}} },
      edges:{ scaling:{min:1,max:8}, color:{color:'#cbd5e1'} },
      physics:{ barnesHut:{ gravitationalConstant:-3000 } } });

아래 텍스트(가상의 사고 진술서 모음)를 자유롭게 수정하고 ▶ 분석을 누르면 의미연결망이 다시 그려집니다. 실제 기사·민원·인터뷰 텍스트를 붙여 넣어도 작동합니다.

상단 텍스트를 수정하고 ▶ 분석 버튼을 누르세요. 큰 노드 = 빈도 높은 단어, 굵은 선 = 자주 함께 등장한 단어 쌍.
