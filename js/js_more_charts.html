⭐ 워드클라우드 — 텍스트의 단어 빈도 시각화 (d3-cloud)
📚 워드클라우드란?
🎯 목적 · 긴 텍스트(기사·인터뷰·SNS·민원)에서 "어떤 단어가 자주 나오는지"를 글자 크기로 시각화. 큰 글자 = 자주 등장.
✨ 특징 · 정량적 분석은 아니지만 "한눈에 핵심 키워드"를 잡아내는 데 효과 만점. 회전·색·폰트로 시각적 임팩트.
🚦 교통사고 사례
  • 경찰 사고 진술서 1년치 → "음주" "신호" "야간" "보행자"가 가장 큰 글자로 부각
  • 국민 민원 텍스트 → 정책 우선순위 단서
  • SNS·뉴스에서 "교통사고" 언급 맥락 파악
⚠️ 주의 · 워드클라우드는 정확한 빈도 비교에는 부적합(글자 크기·회전·길이가 인식을 왜곡). 보조용으로 막대 차트와 함께 사용 권장. 또 불용어(은·는·이·가)를 반드시 제거.
// CDN: d3.v7 + d3.layout.cloud
const layout = d3.layout.cloud()
    .size([W, H])
    .words(words.map(d => ({ text:d.word, size:scale(d.count) })))
    .padding(4).rotate(() => ~~(Math.random()*2)*90)
    .font('Pretendard, "Malgun Gothic"')
    .fontSize(d => d.size)
    .on('end', draw);
layout.start();
Sankey — 흐름(flow) 시각화
📚 Sankey 다이어그램이란?
🎯 목적 · "A에서 B로, B에서 C로" 같은 단계별 흐름의 양을 두께로 시각화. 합계 보존(들어온 만큼 나감)이 핵심 가치.
✨ 특징 · 좌→우 또는 상→하로 흐름. 각 단계에서의 분기·합류·손실이 자연스럽게 보임.
🚦 교통사고 사례
  • "사고유형 → 시도 → 결과(경상/중상/사망)" 3단계 흐름
  • "가해 차종 → 피해 보행자/차량 → 처분(불기소/벌금/실형)"
  • "보험 청구 → 평가 → 합의/소송 → 종결" 같은 프로세스
  • 에너지·예산·인구 이동 같은 비교통 분야에도 광범위 사용
⚠️ 주의 · 단계가 5개 넘으면 시각이 복잡해짐. 사이클(A→B→A)은 표현 불가.
// 사고유형 → 시도 → 결과 흐름
new Chart(ctx, {
    type: 'sankey',
    data: { datasets: [{
        data: [
            { from:'차대차', to:'서울', flow:25800 },
            { from:'서울',   to:'경상', flow:15000 }
        ],
        colorFrom: c => '#3b82f6',
        colorTo:   c => '#ef4444'
    }]}
});
Calendar Heatmap — 일자별 강도 (GitHub 스타일)
📚 Calendar Heatmap이란?
🎯 목적 · 1년치 일별 데이터를 요일 × 주(week) 격자로 펼쳐 색 농도로 표현. GitHub의 contribution graph가 대표 사례.
✨ 특징 · 시계열의 주기성·계절성·요일효과·연휴효과가 한 화면에서 동시에 드러남. 365일을 한눈에.
🚦 교통사고 사례
  • "금/토요일이 다른 요일보다 짙다" → 주말 음주 사고 효과
  • "여름·연말이 더 짙다" → 휴가철·연휴 집중
  • "특정 일자에만 폭증" → 폭우·폭설 같은 외부 사건 추적
⚠️ 주의 · 색 단계(범위)를 잘 잡지 않으면 모든 칸이 비슷해 보임 → 데이터 분포에 맞춰 구간 조정 필수.
new ApexCharts(el, {
    chart: { type: 'heatmap', height: 280 },
    series: weekData,
    plotOptions: { heatmap: {
        colorScale: { ranges: [
            { from:0, to:40,  color:'#fdf2f8', name:'한산' },
            { from:41,to:70,  color:'#fbcfe8', name:'보통' },
            { from:71,to:100, color:'#ec4899', name:'많음' }
        ]}
    }}
});
Sunburst — 계층 비율 (Plotly)
📚 Sunburst란?
🎯 목적 · 트리맵과 비슷하지만 원형 방사 구조. 안쪽 원이 상위 카테고리, 바깥 원이 하위. 클릭으로 줌인 가능.
✨ 특징 · 계층 깊이를 깔끔히 표현. 면적 = 비율. 인터랙티브 탐색에 좋음.
🚦 교통사고 사례
  • "권역(수도권·영남·호남·중부) → 시도 → 사고유형" 3계층 구조
  • "가해자 연령대 → 면허 종류 → 위반항목"
  • "차종 → 연식 → 사고원인" 다층 비교
⚠️ 주의 · 4계층 넘으면 바깥 원이 잘 안 보임. 색상 스케일을 카테고리에 맞춰 잘 골라야 함.
// 계층은 labels + parents 쌍으로 표현
Plotly.newPlot('sunburst', [{
    type: 'sunburst',
    labels:  ['전국','수도권','영남','서울','경기','부산'],
    parents: ['',  '전국','전국','수도권','수도권','영남'],
    values:  [191418, 107170, 47932, 34215, 58432, 15621],
    branchvalues: 'total'
}], { margin: { t:0, l:0, r:0, b:0 } });
💡 조각을 클릭하면 그 그룹만 확대됩니다. 가운데 클릭으로 복귀.
Parallel Coordinates — 다변수 동시 비교 (Plotly)
📚 Parallel Coordinates란?
🎯 목적 · 4~10개의 변수(축)를 평행하게 늘어놓고, 각 데이터(시도 등)가 모든 축을 통과하는 선으로 표현. 다변수 분포·이상치·클러스터 탐색에 강력.
✨ 특징 · 각 축을 드래그하면 그 범위만 필터링. 색은 또 다른 변수(예: 사망률)로 매핑해 패턴 강조.
🚦 교통사고 사례
  • 17개 시도의 (사고건수·사망자·사망률·10만명당) 4축 동시 비교 → "건수는 적은데 사망률은 높은" 시도 발견
  • 차종별 (평균속도·중상비율·연식·연료종류) 다변수 패턴
  • 운전자 프로파일(나이·경력·과실비율·재발률) 클러스터 탐색
⚠️ 주의 · 축 순서에 따라 패턴이 달라 보임. 단위가 매우 다르면 정규화 권장.
Plotly.newPlot('parallel', [{
    type: 'parcoords',
    line: { color: dataValues, colorscale: 'Viridis', showscale: true },
    dimensions: [
        { label:'사고건수', values:[...] },
        { label:'사망자',   values:[...] },
        { label:'사망률',   values:[...] },
        { label:'10만명당', values:[...] }
    ]
}]);
매트릭스 히트맵 — 시간대 × 요일 (chartjs-chart-matrix)
📚 매트릭스 히트맵이란?
🎯 목적 · 두 범주형 변수(요일 × 시간대 등) 교차로 만들어진 격자에 색 농도로 값을 표현. 두 차원의 패턴을 동시 인식.
✨ 특징 · 표보다 패턴이 잘 보임. 흰색~짙은색 그라디언트로 가독성↑.
🚦 교통사고 사례
  • "금요일 오후 6–24시"가 가장 짙음 → 퇴근+불금 효과
  • 월별 × 시도별 사고 매트릭스 → 특정 지역의 계절성
  • 차종 × 위반항목 교차표 → "이륜차는 신호위반이 압도적" 같은 발견
⚠️ 주의 · 숫자 정확도는 떨어짐. 표시값을 칸 안에 적거나 툴팁으로 보완.
new Chart(ctx, {
    type: 'matrix',
    data: { datasets: [{
        data: [{x:'월', y:'0–6시', v:120}, ...],
        backgroundColor: c => `rgba(236,72,153,${c.raw.v/maxV})`,
        width:  ({chart}) => chart.chartArea.width  / 7 - 2,
        height: ({chart}) => chart.chartArea.height / 4 - 2
    }]},
    options: { scales: { x:{type:'category',labels:days}, y:{type:'category',labels:hours} } }
});
박스플롯 — 분포(중앙·사분위·이상치) 비교
📚 박스플롯이란?
🎯 목적 · 평균 한 숫자만 보여주는 막대와 달리 분포 전체(중앙값·25%·75%·이상치)를 한 박스로 요약. 통계 분석의 기본 도구.
✨ 특징 · 박스 = 25~75% 구간(IQR). 가운데 선 = 중앙값. 점 = 이상치. 여러 그룹을 옆에 놓고 비교 가능.
🚦 교통사고 사례
  • 시도별 월별 사고건수 분포 → "경기는 평균도 높고 변동도 큼"
  • 차종별 부상 정도(0~3 척도) 분포
  • 같은 도로 구간의 일별 사고건수 분포로 "비정상적으로 사고가 많은 날" 탐지
⚠️ 주의 · 일반 독자에게는 낯선 그래프. 중앙값·사분위 의미를 캡션으로 풀어주는 게 좋음.
// 그룹별 분포의 중앙·25%/75%·이상치를 한 박스로
Plotly.newPlot('box', [
    { type:'box', y:서울월별,   name:'서울', boxpoints:'outliers' },
    { type:'box', y:경기월별,   name:'경기' },
    { type:'box', y:부산월별,   name:'부산' }
]);
Gauge / KPI — 핵심 지표를 큼직하게 강조
📚 Gauge / KPI 카드란?
🎯 목적 · "올해 사망자 2,551명, 작년 대비 -184명" 같은 핵심 단일 지표를 강하게 부각. 대시보드 상단에 흔히 배치.
✨ 특징 · 숫자 + 변화량(델타) + 게이지. 구간 색칠(녹색·노란색·빨강)로 상태(정상·경계·위험)를 즉시 인식.
🚦 교통사고 사례
  • 실시간 도로 안전 대시보드: 오늘 사고 건수, 사망자, 부상자, 사망률
  • 경찰서 종합 KPI: 단속 건수·검거율·민원 처리 시간
  • 목표 대비 현재 위치: "사망자 2,000명 이하 목표" 게이지
⚠️ 주의 · 숫자 한 개의 임팩트가 너무 크므로 맥락(전년·평균·목표)을 함께 보여줘야 오해를 줄임.
Plotly.newPlot('gauge', [{
    type: 'indicator',
    mode: 'gauge+number+delta',
    value: 191418,
    delta: { reference: 196836 },
    gauge: { axis:{ range:[0, 250000] },
             steps:[{range:[0,200000],color:'lightgreen'},{range:[200000,250000],color:'tomato'}] }
}]);
Funnel — 단계별 감소 시각화
📚 Funnel(깔때기) 차트란?
🎯 목적 · 단계가 진행될수록 대상 수가 줄어드는 과정을 깔때기 모양으로. 각 단계의 통과율·이탈률을 직관적으로.
✨ 특징 · 마케팅의 전환 깔때기(방문→가입→구매)에서 유래. 각 단계 라벨에 절대값 + 첫 단계 대비 % 동시 표시 가능.
🚦 교통사고 사례
  • "신고된 사고 → 경찰 출동 → 부상 → 중상 → 사망" 단계별 감소
  • "위반 적발 → 입건 → 기소 → 유죄" 형사 처리 과정
  • "교육 신청 → 출석 → 이수 → 면허 취득" 같은 절차 흐름
⚠️ 주의 · 단계 순서가 명확하고 단조 감소일 때만 효과적. 단계가 다시 늘면 막대 차트가 더 정직.
Plotly.newPlot('funnel', [{
    type: 'funnel',
    y: ['신고된 사고','경찰 출동','부상 발생','중상 이상','사망'],
    x: [191418, 178520, 134128, 28430, 2551],
    textinfo: 'value+percent initial'
}]);
