DataTables.js — 정렬·검색·페이지네이션이 되는 표

🎯 이 탭에서 배우는 것

"수십 행짜리 표"를 독자가 직접 정렬·검색·페이지 이동할 수 있게 만드는 방법. 기존 HTML <table>에 한 줄만 적용하면 즉시 인터랙티브 테이블로 변신합니다.

📚 DataTables란?
🎯 목적 · 차트로 보여주기엔 항목이 너무 많을 때(시도 17개·차종 30개 등) "독자가 직접 탐색하는 표"로 풀어내는 도구.
✨ 특징 · 헤더 클릭 → 정렬, 검색창 → 즉시 필터, 하단 → 페이지 이동. 서버 없이도 완전 동작.
🚦 교통사고 사례
  • 17개 시도 사고 통계 표 — 사망률 기준 정렬해 위험 지역 찾기
  • 1000건짜리 사망사고 원본 데이터 → 검색으로 특정 운전자/차량 추적
  • 월별·차종별 누적 표 — 페이지로 분할해 가독성 유지
⚠️ 주의 · 데이터가 만 행 넘으면 클라이언트 처리 느려짐 → 서버 사이드 처리로 전환.
// ── DataTables (jQuery 필요) ──
$('#테이블id').DataTable({
    language: {
        search:'검색:', lengthMenu:'_MENU_ 개씩 보기',
        info:'_START_–_END_ / _TOTAL_ 개',
        paginate: { previous:'이전', next:'다음' }
    },
    order: [[1, 'desc']],   // [열 인덱스, 'asc'|'desc']
    pageLength: 7
});

Intersection Observer — "스크롤 감지"의 원리 익히기

🎯 이 탭에서 배우는 것

스크롤 기반 시각화의 기초 원리를 직접 눈으로 확인합니다. 어떤 요소가 화면(또는 컨테이너)에 들어왔다 나갔다 하는지를 브라우저가 어떻게 감지하는지를 보여주는 데모입니다. 이 기술 위에 ③ Scrollama가 만들어졌고, ④ 완성 기사가 동작합니다.

📚 Intersection Observer란?
🎯 목적 · 화면 어딘가에 어떤 요소가 들어왔는지 감지하는 브라우저 기본 API. 옛날엔 scroll 이벤트로 처리했지만 성능이 나빴고, 이걸로 표준화됐습니다.
✨ 특징 · "이 요소가 50% 이상 보이면 콜백 실행" 같은 식. 무한 스크롤·이미지 지연 로드·스크롤 애니메이션·스크롤리텔링의 기반.
🚦 교통사고 사례
  • 아래 7개 박스를 스크롤하면 화면에 들어오는 박스가 보라색으로 활성 → "이 단계에 도달했다"를 감지하는 원리 그대로
  • 독자가 사고 통계 섹션에 도달하면 차트 자동 그려넣기
  • 긴 인포그래픽의 어느 위치인지 우측 진행바에 표시
⚠️ 주의 · 컨테이너(root)가 자체 스크롤되는 경우엔 root: containerEl을 명시. 안 그러면 페이지 스크롤 기준이 됨.
const obs = new IntersectionObserver(
    entries => entries.forEach(e => e.target.classList.toggle('visible', e.isIntersecting)),
    {
        root: containerEl,   // null이면 브라우저 뷰포트
        threshold: 0.5       // 50% 이상 보일 때 발동
    }
);
document.querySelectorAll('.io-item').forEach(el => obs.observe(el));

▼ 아래 박스를 스크롤해 보세요. 화면 안에 들어오면 보라색으로 변합니다.

① 첫 섹션 — 스크롤하면 활성화
② 두 번째 — 화면에 들어오면 보라색
③ 세 번째 — IntersectionObserver 감지
④ 네 번째 — threshold: 50%
⑤ 다섯 번째 — Scrollama의 기반 기술
⑥ 여섯 번째 — 추가 확인용
⑦ 일곱 번째 — 끝
스크롤하면 감지 상태가 표시됩니다
⭐ Scrollama 인라인 데모 — 스크롤로 차트가 바뀐다

🎯 이 탭에서 배우는 것

NYT·워싱턴포스트·SBS·한겨레 등이 쓰는 "스크롤리텔링"의 핵심 패턴.
왼쪽 차트를 화면에 고정(sticky)해 두고, 오른쪽 텍스트를 스크롤하면 단계마다 차트가 바뀌는 구조입니다. 이 한 패턴만 익히면 데이터 기사·내러티브 시각화를 만들 수 있습니다.

📚 Scrollama 패턴이란?
🎯 목적 · "독자가 스크롤할수록 이야기와 시각화가 함께 진행되는" 데이터 저널리즘 표준 형식. 텍스트와 차트를 같이 따라가게 만듦.
✨ 특징 · IntersectionObserver를 한 줄로 감싼 라이브러리. onStepEnter 콜백에서 차트만 바꿔주면 끝.
🚦 교통사고 사례 — 5단계 시나리오
  • SECTION 01: 8년 추세 선그래프 → 감소세 보여주기
  • SECTION 02: 특정 한 해만 빨간 막대로 강조 → 외부 충격(예: 코로나)
  • SECTION 03: 도넛으로 사고유형 비율 전환
  • SECTION 04: 시도별 가로막대로 지역 격차 강조
  • SECTION 05: "2,551명" 숫자 하나만 거대하게 → 핵심 메시지
⚠️ 주의 · 작은 컨테이너 안에서 동작시키면 IntersectionObserver가 어색해짐 → 페이지 자체를 스크롤하는 게 가장 안정적(이 데모도 그 방식).
// 핵심: 페이지 자체를 스크롤 → IO 가장 안정적
const scroller = scrollama();
scroller
    .setup({ step: '.scroll-step', offset: 0.5 })   // step이 화면 50% 지점에 오면 발동
    .onStepEnter(({ element, index }) => {
        document.querySelectorAll('.scroll-step').forEach(s => s.classList.remove('is-active'));
        element.classList.add('is-active');
        showStage(index);   // 차트/숫자 전환
    });
window.addEventListener('resize', scroller.resize);

⭐ 완성 기사 — 풀스크린 데이터 저널리즘 결과물

🎯 이 탭에서 배우는 것

지금까지 배운 모든 것(차트 + 스크롤 감지 + sticky 레이아웃)을 합쳐 만든 실제 신문 기사 형태의 5섹션 인터랙티브 기사입니다.
탭이 아닌 전체 화면으로 봐야 sticky·스크롤이 자연스럽게 동작하므로 새 창으로 엽니다.

📚 완성 기사의 구조
🎯 목적 · 데이터 저널리즘의 표준 양식 — "히어로 → 도입글 → 5섹션 시각화 → 풋터". 독자는 스크롤만 하면 자연스럽게 1년 데이터 분석을 따라가게 됨.
✨ 특징 · position:sticky로 차트 영역이 화면 절반을 고정. 오른쪽 텍스트가 흐르며 차트가 단계 전환. 모바일에서는 차트가 위, 텍스트가 아래로 자동 재배치 가능.
🚦 5섹션 구성
  • ① 8년 연속 감소 추세 (이중축 선그래프)
  • ② 외부 충격 한 해 강조 (강조 막대)
  • ③ 차대차 74% 비중 (도넛)
  • ④ 수도권 집중 65% (가로 막대)
  • ⑤ "하루 7명" 핵심 수치 (큰 숫자 한 줄)
⚠️ 주의 · 일부 브라우저는 새 창에서 document.write를 막을 수 있음 → 그땐 별도 .html 파일로 저장 후 직접 여세요.
// 새 창에 HTML 통째로 작성하는 패턴
function openArticle() {
    const html = `<!DOCTYPE html>...<script>scrollama().setup(...).onStepEnter(...);</script>...`;
    const w = window.open('', '_blank');
    w.document.write(html);
    w.document.close();
}
💡 직접 만들 때는 같은 폴더에 article.html로 저장하고 Live Server에서 열면 됩니다.
