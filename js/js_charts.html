📊 Chart.js — 인터랙티브 차트 종합 가이드

교통사고 데이터(TAAS)로 배우는 9가지 차트 + ⭐인터랙티브 필터

① Bar
② Line (이중축)
③ Pie / Doughnut
④ Radar
⑤ Bubble
⑥ Scatter
⑦ Polar Area
⑧ Mixed (Bar+Line)
⑨ Treemap
⑩ 인터랙티브 필터 ⭐
막대 그래프 (Bar) — 연도별 사고건수
📚 막대 그래프란?
🎯 목적 · 여러 항목의 양(量)을 비교하는 가장 기본적인 차트. 막대 길이가 곧 값.
✨ 특징 · 카테고리 간 차이를 한눈에 인식하기 쉬움. 숫자 정확도가 높음(시작점이 0이므로 비례 인식 직관적).
🚦 교통사고 사례
  • 연도별 사고건수처럼 같은 단위·같은 측정값의 시간 비교
  • 시도별 사망자처럼 지역 간 순위·격차 비교
  • 차종별·연령대별 빈도처럼 범주별 카운트
⚠️ 주의 · 항목이 너무 많으면(20개+) 가로 스크롤이 필요해 가독성 ↓ — 그땐 가로 막대(indexAxis:'y')나 트리맵을 고려.
// ── Chart.js 기본 3단계 ──
// 1) canvas 요소 선택
const ctx = document.getElementById('barChart');

// 2) Chart 인스턴스 생성
new Chart(ctx, {
    type: 'bar',
    data: {
        labels: years,
        datasets: [{ label:'교통사고 건수', data: 사고건수,
            backgroundColor: 'rgba(59,130,246,0.7)' }]
    },
    options: { responsive: true, maintainAspectRatio: false }
});
💡 maintainAspectRatio:false를 켜야 부모 div의 height에 맞춰 늘어납니다.
선 그래프 (Line) — 사고건수 + 사망자 (이중 축)
📚 선 그래프(특히 이중축)란?
🎯 목적 · 시간에 따른 변화 추세를 보여주는 데 최적. 점들을 연결한 선이 "흐름"을 인식시킴.
✨ 특징 · 이중축(좌·우 다른 스케일)을 쓰면 단위가 다른 두 시리즈를 한 화면에서 비교 가능.
🚦 교통사고 사례
  • "사고건수(만 단위) ↔ 사망자(명 단위)"처럼 스케일이 10000:1로 다른 두 지표를 같이 보고 싶을 때
  • "통행량 ↔ 사고건수" 추세 비교, "강수량 ↔ 야간 사고" 동행성 확인
  • "코로나 전·후" 같은 시계열 변곡점 강조
⚠️ 주의 · 이중축은 인과를 시사하지 않음 — 두 선이 "같이 움직이는 것"일 뿐. 또 양쪽 스케일 시작점을 임의로 잡으면 시각적 왜곡 가능.
// 단위가 다른 두 시리즈를 한 차트에 → yAxisID + scales 분리
datasets: [
    { label:'사고건수(만)', data:..., yAxisID:'y',  borderColor:'#3b82f6', fill:true },
    { label:'사망자(명)',   data:..., yAxisID:'y2', borderColor:'#ef4444' }
],
scales: {
    y:  { position:'left'  },
    y2: { position:'right', grid:{ drawOnChartArea:false } }
}
파이 / 도넛 — 사고유형별 비율
📚 파이/도넛 차트란?
🎯 목적 · 전체 100% 중 부분 비율을 직관적으로 표현. "차대차가 전체의 74%"처럼.
✨ 특징 · 파이는 꽉 찬 원, 도넛은 가운데가 비어 있어 합계·핵심 수치를 글자로 넣기 좋음.
🚦 교통사고 사례
  • 사고유형별 비율(차대차·차대사람·차량단독)
  • 월별 사고가 아닌 월별 사망 비율처럼 합이 의미 있는 항목
  • 가해 차종 비율(승용·승합·화물·이륜)
⚠️ 주의 · 항목이 5개 넘으면 각도 차이가 안 보임 → 막대가 더 나음. 시계열 비교에도 부적합(파이를 여러 개 늘어놓는 건 비효율).
// type만 'pie' ↔ 'doughnut' 으로 바꾸면 됩니다
new Chart(ctx, {
    type: 'pie',  // 또는 'doughnut'
    data: { labels:[...], datasets:[{ data:[...], backgroundColor:[...] }] },
    options: { plugins:{ tooltip:{ callbacks:{
        label: c => `${c.label}: ${(c.raw/total*100).toFixed(1)}%`
    }}}}
});
레이더 (Radar) — 시도별 다차원 비교
📚 레이더 차트란?
🎯 목적 · 한 대상의 여러 항목 점수를 동시에 보여주는 다각형 프로파일. 모양 자체가 "특성"을 시각적 패턴으로 만듦.
✨ 특징 · 여러 다각형을 겹쳐 그려 대상 간 패턴 차이를 비교. 각 꼭짓점은 한 항목.
🚦 교통사고 사례
  • 각 시도의 (사고건수·사망자·부상자·사망률·10만명당) 5개 지표 동시 비교
  • 운전자 프로파일: (반응속도·시야·집중력·경험·피로도)
  • 도로 안전도: (가로등·신호·횡단보도·과속단속·CCTV) 다항 점수
⚠️ 주의 · 항목 단위가 모두 달라서 그대로 그리면 큰 값 항목이 지배 → 0~1 정규화가 필수. 다각형이 3개 넘으면 겹쳐서 안 보임.
// labels = 꼭짓점, datasets = 비교 대상 (여러 다각형)
data: {
    labels: ['서울','경기','부산','대구','인천','광주','대전'],
    datasets: [
        { label:'사고건수(백)', data:[342,584,156,98,145,72,81] },
        { label:'사망자',       data:[252,681,178,98,141,67,83] }
    ]
}
버블 차트 — 사고건수(x) × 사망자(y) × 인구(원 크기)
📚 버블 차트란?
🎯 목적 · 산점도에 "세 번째 변수"를 원의 크기로 추가. 한 차트에 3변수를 동시 표현.
✨ 특징 · 위치(x,y) + 크기(r). 색상까지 쓰면 4변수도 가능. "고위험 지역"을 큰 빨간 원처럼 강조 가능.
🚦 교통사고 사례
  • x=사고건수, y=사망자, r=인구 → "인구 대비 사고가 많은 지역"이 한눈에
  • x=주행거리, y=사고율, r=차령(차량 나이) → 노후 차량 위험도
  • x=음주측정 횟수, y=적발률, r=면허 보유자 수
⚠️ 주의 · r은 면적이 값에 비례하도록 √(값) 변환을 권장. 직접 r=값으로 하면 큰 값이 과장됨.
// 각 점이 {x, y, r} 객체. r은 픽셀 단위 반지름
datasets: [{
    label:'서울',
    data:[{ x:342, y:252, r:18 }],
    backgroundColor:'rgba(59,130,246,0.7)'
}]
산점도 (Scatter) — 사고건수와 사망률의 상관
📚 산점도란?
🎯 목적 · 두 변수의 관계(상관·역상관·무관)를 점들의 분포 패턴으로 확인. 통계 분석의 출발점.
✨ 특징 · 점이 우상향이면 양의 상관, 우하향이면 음의 상관, 흩어져 있으면 무관. 회귀선·이상치도 한눈에.
🚦 교통사고 사례
  • x=사고건수, y=사망률(%) → "사고가 많은 곳이 더 위험한가?"
  • x=차량 평균속도, y=중상 비율 → 속도-피해 정도의 비선형 관계
  • x=음주운전 단속 건수, y=음주 사고 → 단속 효과 분석
⚠️ 주의 · 산점도가 보여주는 건 상관(correlation)일 뿐 인과(causation)가 아닙니다. 해석 신중히.
// type:'scatter', data는 {x, y} 배열
new Chart(ctx, {
    type: 'scatter',
    data: { datasets: [{
        label:'시도', data:[{x:34215,y:0.74},{x:58432,y:1.17},...],
        backgroundColor:'rgba(99,102,241,0.7)'
    }]},
    options: {
        scales:{ x:{ title:{display:true,text:'사고건수'} },
                 y:{ title:{display:true,text:'사망률(%)'} } }
    }
});
Polar Area — 시간대별 사고 분포
📚 Polar Area 차트란?
🎯 목적 · 파이와 비슷하지만 각도는 모두 같고 반지름이 값을 표현 → 카테고리 수를 즉시 인식하면서 양도 비교.
✨ 특징 · 시각적으로 임팩트가 강함. 빈도가 큰 시간대가 멀리 뻗어나가는 형태로 보여 "어디가 위험한지" 즉각 인식.
🚦 교통사고 사례
  • 시간대별 사고(주중주간·야간·주말·심야): 5~7개 카테고리에 적합
  • 요일별 사고 분포(월~일 7개)
  • 월별 강수일 vs 사고 빈도
⚠️ 주의 · 반지름이 값이지만 면적은 r²에 비례해 시각적 과장이 일어남. 정밀 비교가 목적이면 막대가 더 정확.
// pie와 비슷하지만 각도는 균등하고 반지름이 값
new Chart(ctx, {
    type: 'polarArea',
    data: {
        labels: ['주중 주간','주중 야간','주말 주간','주말 야간','심야'],
        datasets: [{ data:[78532,42180,38420,28910,9376], backgroundColor:[ ... ] }]
    }
});
혼합 차트 (Mixed) — 막대(건수) + 선(사망률)
📚 혼합(Mixed) 차트란?
🎯 목적 · "양"과 "비율"처럼 성격이 다른 두 지표를 한 차트에서 동시에 보여줌. 막대=양, 선=추세.
✨ 특징 · 데이터 시리즈마다 type을 다르게 줘서 자동으로 합쳐짐. 이중축과 결합하면 더욱 강력.
🚦 교통사고 사례
  • 막대=사고건수, 선=사망률(%) → "건수는 줄지만 치사율은 그대로"가 보임
  • 막대=월별 단속 건수, 선=음주 사고 비율
  • 막대=신호 위반, 선=평균 벌금
⚠️ 주의 · 시리즈가 3개 넘으면 색·축이 헷갈림. 핵심 2개만 강조.
// dataset마다 type을 다르게 → 자동으로 mixed
type: 'bar',
data: { datasets: [
    { type:'bar',  label:'건수', data:[...], yAxisID:'y' },
    { type:'line', label:'사망률(%)', data:[...], yAxisID:'y2',
      borderColor:'#ef4444', tension:0.3 }
]}
트리맵 (Treemap) — 시도 × 사고유형 계층
📚 트리맵이란?
🎯 목적 · "부분/전체 비율"을 사각형 면적으로 + "계층 구조"를 그룹 박스로 표현. 한 화면에 수십~수백 항목 표시 가능.
✨ 특징 · 면적이 곧 양. 색상으로 카테고리 구분. 항목이 많아도 일관되게 표현 가능.
🚦 교통사고 사례
  • "권역 → 시도 → 사고유형" 3계층 비율
  • "차종 → 연식 → 사고 빈도"
  • "가해 연령대 → 면허 종류 → 사고 비율"
⚠️ 주의 · 사각형 비교는 면적 인식이 길이 인식보다 약함. 정확한 순위 비교는 막대가 우수.
// 별도 플러그인 CDN 필요
new Chart(ctx, {
    type: 'treemap',
    data: { datasets: [{
        tree: rawTree,             // [{ 시도, 유형, 건수 }, ...]
        key: '건수',                // 면적 기준
        groups: ['시도','유형'],    // 계층 순서
        backgroundColor: c => c.type==='data' ? colorByGroup(c.raw._data.시도) : 'transparent',
        labels: { display:true, color:'white' }
    }]}
});
⭐ 인터랙티브 필터 — 드롭다운 → 차트 실시간 업데이트
📚 인터랙티브 차트의 가치
🎯 목적 · 독자가 직접 지표·차트유형·기간을 바꿔가며 데이터를 "탐색"할 수 있게 만들기. 정적 차트와의 결정적 차이.
✨ 핵심 패턴 2가지
  • 데이터·색만 변경: chart.data... = ...; chart.update(); (애니메이션 자연스러움)
  • 차트 종류 변경: chart.destroy() → 새 new Chart() 생성 (Chart.js 4의 정석)
🚦 교통사고 사례
  • 독자: "사망자 추이만 보고 싶다" → 지표 드롭다운으로 즉시 전환
  • "막대보다 선이 더 직관적" → 차트유형 토글
  • "최근 3년만 보고 싶다" → 슬라이더로 범위 조정
⚠️ 주의 · UI 옵션을 너무 많이 두면 오히려 길을 잃음. 핵심 변수 2~3개만.
// ── 핵심 패턴 ──
// (1) 데이터·색만 바뀌면 → chart.data 변경 후 chart.update()
// (2) 차트 type이 바뀌면 → destroy() 후 새 Chart() 재생성

let chart = makeChart(currentType, currentData);

select.addEventListener('change', () => {
    if (typeChanged) {
        chart.destroy();                 // ← 핵심
        chart = makeChart(newType, newData);
    } else {
        chart.data.datasets[0].data = newData;
        chart.update();
    }
});
