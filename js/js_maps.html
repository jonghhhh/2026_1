Leaflet 기초 — 지도 + 마커 + 팝업
📚 지도 시각화의 출발점
🎯 목적 · 데이터에 위도/경도가 있다면 가장 먼저 떠올릴 것은 "지도에 점 찍기". "어디서?"라는 질문에 가장 직관적인 답.
✨ 특징 · Leaflet은 무료 오픈소스이자 가장 가벼운 지도 라이브러리. 3단계(지도생성 → 타일 → 마커)로 끝남.
🚦 교통사고 사례
  • 특정 사망사고 5건의 위치를 마커로 → "어느 교차로?"
  • 학교 주변 어린이보호구역 위반 지점 표시
  • 음주운전 단속 지점 + 시간대 팝업
⚠️ 주의 · 점이 100개 넘어가면 가독성 폭락 → 클러스터(②) 또는 히트맵(④) 추천.
// ── 3단계 패턴 ──
const map = L.map('map').setView([37.5665, 126.978], 11);
L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', { attribution:'© OpenStreetMap' }).addTo(map);
L.marker([37.5665, 126.978]).bindPopup('<b>서울시청</b><br>사고 45건').addTo(map);
⚠️ <div id="map">에 반드시 height가 지정돼 있어야 합니다 (없으면 0px → 보이지 않음).
마커 클러스터 — 다수 데이터를 자동 그룹화
📚 마커 클러스터링이란?
🎯 목적 · 점이 수백~수천 개가 되면 화면이 까매짐 → 가까운 점들을 "숫자가 적힌 동그라미 하나"로 묶어 보여줌. 줌인하면 자동 분리.
✨ 특징 · 데이터 양과 무관하게 화면이 깔끔. 색·필터로 카테고리별 토글 가능.
🚦 교통사고 사례
  • 한 해 서울 사고 30,000건을 그대로 점찍기 → 클러스터로 자동 정리
  • 유형(차대차/차대사람/차량단독)별 색상 + 체크박스로 켜고 끄기
  • 112 신고 데이터 5,000건 시각화 (실시간 운영도 가능)
⚠️ 주의 · 클러스터는 밀도를 보여주지만 총량은 직관적으로 안 보임 → 총량 비교는 코로플레스(⑤)가 적합.
const cluster = L.markerClusterGroup();
data.forEach(d => {
    L.circleMarker([d.lat, d.lng], { radius:6, fillColor:색상[d.type], color:'#fff', fillOpacity:0.8 })
     .bindPopup(d.type).addTo(cluster);
});
map.addLayer(cluster);  // 줌아웃 → 묶임, 줌인 → 펼침

원형 마커(Proportional Symbol Map) — 반지름으로 정량 표현
📚 정량 원형 마커란?
🎯 목적 · 위치 + "얼마나"를 동시에. 원의 크기가 곧 값(사고 건수, 사망자 수 등).
✨ 특징 · 대륙·도시 단위처럼 지점 데이터의 양적 차이를 한눈에. 위치-양 결합 시각화의 표준.
🚦 교통사고 사례
  • 주요 교차로 7곳의 사고건수: 강남역(850) > 잠실(680) > 서울역(520)…
  • 지하철역별 보행 사고 빈도 (작은 원→큰 원)
  • 시도 중심점에 인구 대비 사망률 원으로 표시
⚠️ 주의 · 면적이 값에 비례해야 정확 → radius = √(값)로 변환. 직접 r=값으로 하면 큰 값이 4배·9배 과장됨.
data.forEach(d => {
    L.circleMarker([d.lat, d.lng], {
        radius: Math.sqrt(d.건수) / 1.6,   // ← √ 변환이 핵심
        fillColor: '#ef4444', color:'white', weight:2, fillOpacity:0.65
    }).bindPopup(`${d.이름}: ${d.건수}건`).addTo(map);
});
히트맵 — 사고 밀집 지역 시각화
📚 히트맵이란?
🎯 목적 · 점들이 너무 많을 때, "어디에 얼마나 모여 있는가"를 색의 농도로 보여줌. 패턴(핫스팟)이 즉시 보임.
✨ 특징 · 개별 점은 안 보이지만 전체적 밀도 분포가 그림처럼 드러남. 색 그라디언트로 강도 표현.
🚦 교통사고 사례
  • "서울 어디가 사고 핫스팟인가?" — 강남·잠실·광화문 일대가 빨갛게
  • 야간/주말 한정 데이터로 만들면 시간대별 위험 구역 차이 발견
  • 음주 사고만 필터해 시각화 → 유흥가 주변 빨강
⚠️ 주의 · 정확한 수치 비교는 불가(시각적 인상). 보조로 마커·툴팁을 같이 쓰는 게 좋음.
// 데이터 형식: [[위도, 경도, 강도(0~1)], ...]
L.heatLayer(heatPoints, {
    radius: 22, blur: 15, maxZoom: 14,
    gradient: { 0.2:'blue', 0.5:'lime', 0.8:'yellow', 1:'red' }
}).addTo(map);

코로플레스(Choropleth) — 행정구역 색상 단계구분
📚 코로플레스란?
🎯 목적 · 행정구역(시도/시군구/동) 단위 통계를 구역의 색 농도로 표현. "어느 지역이 가장 위험한지" 즉시 인식.
✨ 특징 · 마우스 hover로 강조, 클릭 팝업으로 정확 수치 확인. 인구 대비 같은 비율 지표와 잘 맞음.
🚦 교통사고 사례
  • 17개 시도별 사고건수: 경기·서울이 짙은 파랑
  • 인구 10만명당 사망자율 → "절대 수치"가 아닌 "위험도" 비교
  • 시군구 단위 어린이 보호구역 위반 분포
⚠️ 주의 · 행정구역 면적이 클수록 시각적으로 강조됨 → 큰 지역(강원·경북)이 과대평가될 위험. 인구 가중 지표 사용 권장.
function getColor(v) {
    return v > 50000 ? '#084594' : v > 30000 ? '#2171b5' :
           v > 15000 ? '#4292c6' : v > 10000 ? '#6baed6' :
           v >  5000 ? '#9ecae1' : '#deebf7';
}
L.geoJSON(geojson, {
    style: f => ({ fillColor:getColor(data[f.properties.name]||0), fillOpacity:0.7, color:'#fff', weight:1 }),
    onEachFeature: (f, layer) => {
        layer.bindPopup(f.properties.name);
        layer.on('mouseover', () => layer.setStyle({ fillOpacity:0.95, weight:2 }));
        layer.on('mouseout',  () => layer.setStyle({ fillOpacity:0.7, weight:1 }));
    }
}).addTo(map);
💡 실제 시도 GeoJSON(cdn.jsdelivr.net/gh/southkorea/southkorea-maps)을 받아와 17개 시도 경계 그대로 단계구분도를 그립니다. 인터넷이 차단된 환경에서는 자동으로 사각형 폴백이 동작합니다.
폴리라인 / 폴리곤 / 원 — 경로와 영역
📚 선·면 시각화란?
🎯 목적 · 점이 아닌 "길/구역/반경"을 그리고 싶을 때. 폴리라인=선, 폴리곤=닫힌 면, 원=반경.
✨ 특징 · GPS 트랙·관심 영역(AOI)·반경 원(영향 범위)을 표현 가능.
🚦 교통사고 사례
  • 운전자 사고 직전 GPS 경로 (폴리라인)
  • 어린이 보호구역 다각형 영역 (폴리곤) + 위반 마커
  • 학교 반경 300m 원 표시 → 안전 점검 대상 자동 추출
⚠️ 주의 · 폴리곤은 좌표를 닫힌 순서로 줘야 함. 너무 복잡한 폴리곤은 성능 저하 → simplify 라이브러리 활용.
// 폴리라인 (경로)
L.polyline([[lat1,lng1], [lat2,lng2], ...], { color:'#ef4444', weight:4 }).addTo(map);
// 폴리곤 (닫힌 영역)
L.polygon([[lat,lng], [lat,lng], [lat,lng]], { color:'#3b82f6', fillOpacity:0.3 }).addTo(map);
// 원 (반경, 단위: 미터)
L.circle([37.55, 127.0], { radius:1500, color:'#10b981', fillOpacity:0.2 }).addTo(map);
베이스 타일 전환 + 레이어 컨트롤
📚 레이어 컨트롤이란?
🎯 목적 · 독자가 직접 지도 스타일(밝은/어두운/위성)과 표시 레이어(마커·히트맵)를 켜고 끌 수 있게 만들기.
✨ 특징 · 우상단 컨트롤에서 베이스맵은 라디오, 오버레이는 체크박스로 자동 생성. 한 줄로 끝.
🚦 교통사고 사례
  • 야간 사고 시각화엔 어두운 베이스맵이 더 돋보임
  • "마커만/히트맵만/둘 다" 자유 토글로 독자 탐색
  • 위성 사진과 합쳐 도로·교차로 형태까지 확인
⚠️ 주의 · 베이스맵은 각자 사용 약관 있음(위성은 보통 키 필요). 무료는 OSM·CartoDB 정도.
const osm   = L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png');
const dark  = L.tileLayer('https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}.png');
const light = L.tileLayer('https://{s}.basemaps.cartocdn.com/light_all/{z}/{x}/{y}.png');
L.control.layers(
    { 'OSM': osm, 'Dark': dark, 'Light': light },           // 베이스 (라디오)
    { '마커': markerLayer, '히트맵': heatLayer }            // 오버레이 (체크박스)
).addTo(map);
